# 🚀 Guia para Publicar a Linha Cronológica no GitHub Pages

Este guia irá ajudá-lo a publicar a sua linha cronológica de Portugal online, de forma gratuita e permanente, utilizando o GitHub Pages. Não precisa de ser um programador para seguir estes passos!

## 📋 Pré-requisitos

1.  **Conta GitHub**: Se ainda não tem, crie uma conta gratuita em [github.com](https://github.com/).
2.  **Ficheiros do Projeto**: Certifique-se de que tem a pasta `portugal_timeline` (que lhe forneci) no seu computador, contendo `index.html`, `style.css`, `script.js` e a pasta `images`.

## 📝 Passos para Publicar

### Passo 1: Criar um Novo Repositório no GitHub

1.  Faça login na sua conta GitHub.
2.  No canto superior direito, clique no sinal `+` e selecione **"New repository"**.
3.  **Nome do Repositório**: Dê um nome ao seu repositório, por exemplo, `portugal-timeline`. Este nome fará parte do URL do seu site, por isso, escolha algo simples e sem espaços.
4.  **Descrição (Opcional)**: Adicione uma breve descrição.
5.  **Público/Privado**: Selecione **"Public"** (o GitHub Pages só funciona com repositórios públicos).
6.  **Inicializar**: **NÃO** selecione a opção "Add a README file", "Add .gitignore" ou "Choose a license" por agora. Queremos um repositório vazio.
7.  Clique em **"Create repository"**.

### Passo 2: Carregar os Ficheiros do Projeto

Após criar o repositório, será direcionado para uma página com instruções. Escolha a opção **"uploading an existing file"** (carregar um ficheiro existente).

1.  Arraste e largue todos os ficheiros e pastas do seu projeto (`index.html`, `style.css`, `script.js`, e a pasta `images`) para a área indicada na página do GitHub.
2.  Na parte inferior da página, adicione uma mensagem de "commit" (por exemplo, "Initial commit") e clique em **"Commit changes"**.

### Passo 3: Ativar o GitHub Pages

1.  No seu repositório (ex: `github.com/o-seu-utilizador/portugal-timeline`), clique no separador **"Settings"** (Definições).
2.  No menu lateral esquerdo, clique em **"Pages"**.
3.  Em "Branch", selecione a branch **`main`** (ou `master`, dependendo do nome padrão do seu repositório) e a pasta **`/ (root)`**.
4.  Clique em **"Save"**.

### Passo 4: Aceder ao Seu Site

Após guardar as definições, o GitHub Pages irá demorar alguns minutos a construir e publicar o seu site. Verá uma mensagem a indicar que o seu site está pronto em `https://o-seu-utilizador.github.io/nome-do-repositorio/`.

-   **Exemplo de URL**: Se o seu nome de utilizador for `joao` e o repositório se chamar `portugal-timeline`, o seu site estará em `https://joao.github.io/portugal-timeline/`.

### Passo 5: Atualizar o Conteúdo (Opcional)

Sempre que quiser fazer alterações ao seu site:

1.  Edite os ficheiros no seu computador.
2.  Volte ao seu repositório no GitHub.
3.  Clique em **"Add file"** > **"Upload files"**.
4.  Arraste e largue os ficheiros alterados (ou novos ficheiros) para o repositório. O GitHub irá substituir os ficheiros antigos.
5.  Adicione uma mensagem de "commit" e clique em **"Commit changes"**.

O GitHub Pages irá detetar as alterações e atualizar o seu site automaticamente em poucos minutos.

---

Ao seguir estes passos, terá a sua linha cronológica de Portugal online, permitindo-lhe **explorar os dados de forma mais intuitiva** em qualquer lugar, **compreender melhor as tendências** históricas e **guardar ou partilhar facilmente** este recurso com amigos, família ou colegas. Boa sorte!
